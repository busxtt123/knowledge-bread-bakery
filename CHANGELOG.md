## 2.0.0 — 2026-05-16
- 将知识面包房收束为可独立下载的 skill：根目录直接提供 `SKILL.md`、`references/`、`scripts/` 和版本记录。
- 收束材料入口：只处理本地文档、电子书和已经整理成文件的文本资料；不再抓取或转换网页、公开视频、微信公众号、YouTube 或在线媒体。
- 强化开门清点：所有进入验料和创作的来源必须先落成可读 Markdown；普通 PDF 默认抽取失败时才走 OCR fallback，OCR 失败产物不能当作资料。
- 封装 OCR 环境准备：新增 `scripts/setup-ocr-deps.py`，用于检查并按系统包管理器尝试安装 Poppler / Tesseract / 中文语言包；`ocr-pdf-to-md.py --install-deps` 可以在 OCR 运行前显式准备依赖。
- 强化结构抽取：书籍和电子书优先核对目录；扫描 PDF、OCR 文本或格式异常材料没有可靠目录时，再按语义边界划材料单元。
- 强化 PDF 出柜：`delivery/final.md` 必须用 `# 主标题` 后接 `> 副标题`；标准脚本生成前会拦截缺副标题、标题层级和目录边界问题。
- 将生产现场记录统一为 `workspace/project-log.md`：主理人品尝、创作者必要沟通、回炉请求、故障接管和覆盖恢复都归到这一处。
- 重整委派系统：正常委派回到 `SKILL.md` 主流程和对应工位 SOP，删除独立正常委派侧页；异常只由 `references/subagent-provider-pitfalls.md` 接管。
- 重整材料转换系统：开门清点只写 `sources/source-manifest.md`，删除独立转换备忘录；长期转换经验进入开门清点 SOP 或脚本说明。
- 给边界检查员明确触发点：高影响主题、事实争议、推断边界敏感或主理人拿不准时启用，不把它变成每单必经流程。
- 清理运行缓存：移除 tracked `__pycache__`，并忽略 `__pycache__/` 与 `*.pyc`。
- 保留 1.6.0 已跑通的创作方向：逐讲串行、禅定时刻、soul 与方向、读者认知体验底线、PDF 单一出口。
- 补回 1.5.4 / 1.6.0 的关键生产约束：配方站的原文材料包必须能直接派工，多文件场景不能只写行号；读者认知体验底线改为自然中文表达，写清张力从哪里来，避免用指责、羞耻或亏欠启动阅读。

## 1.6.0 — 2026-05-16
- 收束内容形态：生产系统只判断单篇长文和多讲课程；PDF、Markdown 和讲稿外观只作为交付格式或外部包装，练习只作为课程内部吸收设计。
- 统一分量标准：段落尽量不超过 200 字；阅读速度按 200 字 / 分钟估算；单讲通常 2,080-3,600 中文字，最佳状态约 2,800 字；单篇长文通常 6,000 字以上，超出经验范围必须说明理解收益。
- 修复验料协作边界：结构抽取默认保留完整来源，不为并行把材料切成匿名片段；只有材料总量接近或超过 256k 估算 tokens，或 Markdown 文件数超过 10 个，才启用多同事协作；多文件位置必须写文件名和起止行。
- 重写创作工位：创作者按入场、写作、禅定、交回工作；创作只能逐讲串行推进，不能并行写相邻讲次；创作日记服务创作者连续性，不作为主理人验收依据。
- 将旧“范式异议权”收束为回炉请求：创作者在禅定时刻主动判断 soul、方向、材料现场和读者认知体验；需要返站时写入 `workspace/project-log.md`，不直接覆盖 `workspace/course-plan.md`。
- 明确维护读者认知体验底线：创作和品尝都检查冲突、失效或不安是否来自材料、旧模型、现实机制或结构错位，避免用指责、羞耻或亏欠启动阅读。
- 继续保留单一 PDF 出口，明确标准脚本只依赖 `markdown` 和 `weasyprint`，不依赖 `markdown_it`；前言、发刊词和导论必须保持二级标题。
