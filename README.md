# Knowledge Bread Machine｜知识面包机

把一本书，做成类似得到课程的知识面包。

“知识面包机”吸取了《得到品控手册 9.0》里关于“做内容”的经验，也借用了《哆啦 A 梦》记忆面包的想象：不是把知识压缩成几条笔记，而是把一本书重新做成高营养、易吸收、适合你当前需要的知识课程。

它会先采访你，听清你为什么想读、想用多久读完、希望解决什么问题，再把书里的材料做成一份可以真正吸收的课程。让你在工作、学习和生活都很忙的时候，仍然有心力去接近一本好书、理解一套知识、带走一个能用的判断。

## 它能做什么

- 把一本书做成 30 分钟能读完的知识课程。
- 把复杂资料做成 3-5 讲的小课程。
- 把厚 PDF 做成手机友好的阅读成品。
- 围绕你的问题重组材料，比如“这本书对我的管理判断有什么帮助”。
- 保留知识营养，同时降低进入门槛。

## 怎么开始

把本地文件路径发给 agent，说一句：

```text
请你将 /path/to/book.pdf 这本书转变为课程。
```

也可以更具体：

```text
请使用 knowledge-bread-machine，把 /path/to/book.pdf 做成午休 30 分钟能读完的知识面包。
```

如果你还没想清楚成品形态：

```text
请使用 knowledge-bread-machine，先采访我，再把 /path/to/book.epub 做成适合我阅读的课程。
```

你不需要先写大纲。给它材料，告诉它你的阅读目的，它会继续追问必要信息，然后开始制作。

## 它会经历什么

1. 采访你，确认目标、时间和阅读场景。
2. 读取本地书籍或资料。
3. 提取书的结构和关键内容。
4. 设计课程路线。
5. 写成单篇长文或多讲课程。
6. 交付最终文稿或手机阅读 PDF。

## 适合的材料

- PDF、扫描 PDF、EPUB。
- Word、Markdown、TXT。
- 报告、讲义、课程草稿。
- 已整理成文件的资料包。

它不抓取网页，不转换公开视频、微信公众号、YouTube 或其他在线链接。请先把材料准备成本地文件。

## 需要什么模型

这是长时任务。一本书通常需要 30 分钟到 3 小时，取决于书的体量、课程讲数和模型思考强度。

推荐使用长上下文、长任务稳定、擅长规划和写作的模型，例如 DeepSeek V4 Flash / Pro、Kimi 2.6 等。

## 安装

把仓库地址发给 agent，让它安装这个 skill：

```text
https://github.com/busxtt123/knowledge-bread-machine
```

如果 agent 需要 SSH 地址：

```text
git@github.com:busxtt123/knowledge-bread-machine.git
```

安装后，把材料路径交给 agent，就可以开始做知识面包。

## 扫描 PDF

普通 PDF、EPUB、Word、Markdown、TXT 通常可以直接处理。

扫描 PDF 需要 OCR。PDF OCR 需要本机安装识别工具；如果环境还没准备好，agent 会按提示指导你安装依赖工具。

识别质量太差时，OCR 结果不会进入后续创作。
