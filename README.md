# Knowledge Bread Machine｜知识面包机

把一本厚书、一份报告、一组复杂资料，做成你真的读得下、记得住、用得上的知识面包。

它不是摘要工具。它会先问你为什么想读，再把材料变成适合你的课程、长文或手机 PDF。知识不被削薄，只是被重新切分、发酵、烤熟，变成忙碌生活里也能吸收的一份好内容。

## 适合谁

- 想读一本书，但没有整块时间。
- 想把复杂资料变成可学习的课程。
- 想围绕一个问题读材料，比如“这本书对我的工作判断有什么帮助”。
- 想把 PDF、EPUB、Word、Markdown、TXT 做成手机友好的阅读成品。

## 它会做什么

- 采访你：听清目标、时间、已有理解和阅读场景。
- 处理材料：把本地文档和电子书转成可读文本。
- 设计路线：决定先讲什么、慢讲什么、哪些内容留作背景。
- 创作课程：把材料写成单篇长文或多讲课程。
- 交付成品：输出最终文稿或手机阅读 PDF。

## 一炉需要多久

这是长时任务。一本书通常需要 30 分钟到 3 小时，取决于材料体量、模型思考强度、课程讲数和返工次数。

建议使用长上下文、长任务稳定、擅长规划和写作的模型，例如 DeepSeek V4 Flash / Pro、Kimi 2.6 等。供应商也要稳定，优先选择官方开放平台或可靠服务商，例如 DeepSeek 官方开放平台。

模型越稳定，成品越容易从访谈、验料、配方到创作保持同一口气。

## 怎么唤醒它

把本地文件路径发给 agent，说清你想做成什么。

```text
请你将 /path/to/book.pdf 这本书转变为课程。
```

```text
请使用 knowledge-bread-machine，把 /path/to/book.pdf 做成午休 30 分钟能读完的知识面包。
```

```text
请使用 knowledge-bread-machine，先采访我，再把 /path/to/book.epub 做成适合我阅读的课程。
```

你不需要先写大纲。告诉它材料在哪里、你为什么想读、希望用多久吸收，它会继续追问必要信息，然后开始生产。

## 支持的材料

- PDF、扫描 PDF、EPUB。
- Word、Markdown、TXT。
- 报告、笔记、讲义、课程草稿。
- 已整理成文件的资料包。

它不抓取网页，不转换公开视频、微信公众号、YouTube 或其他在线链接。请先把正文、字幕、转写稿或电子书准备成本地文件。

把仓库地址发给 agent，让它安装这个 skill：

```text

```

如果 agent 需要 SSH 地址：

```text

```

安装后，把材料路径交给 agent，就可以开始做知识面包。

## 扫描 PDF

普通 PDF、EPUB、Word、Markdown、TXT 通常可以直接处理。

扫描 PDF 需要 OCR，也就是识别图片里的文字。PDF OCR 需要本机安装识别工具；如果环境还没准备好，agent 会按提示指导你安装依赖工具。

OCR 结果会经过可读性检查。识别失败或质量太差时，不会被当作可用资料进入后续创作。

## 最终得到什么

- 单篇长文。
- 多讲课程。
- 手机阅读 PDF。
- 必要的过程稿，方便回看材料如何变成课程。

目标不是“内容很多”，而是让你进入材料、抓住主线、理解关键转折，并带走一种更清楚的判断或语言。
