# 20 材料结构抽取方法

## 0. 这一站为什么存在

结构抽取是试材料的劲。摘要回答“材料说了什么”；试材料的劲回答“材料真正能支撑什么”。

这一站先对材料负责，不急着满足顾客想要的形态。材料有自己的主命题、主线、边界和证据链。只有先把这些东西显出来，后面的配方师才知道哪些营养必须保留，哪些材料不能硬塞进正文，创作者也才有可能从材料里醒出一个有灵魂的人。

## 1. 定位边界

材料是否可读，已经由洞见发掘者 SOP 在开工时确认。本页不重复开工检查，只说明结构抽取怎样留下可追溯的位置。

读取 `sources/source-manifest.md` 的目的，是知道每份 Markdown 副本对应哪份原始材料、转换路线是什么、有没有可读边界。随后在结构抽取中为每个材料单元建立更细的位置参数：Markdown 起止行、章节标题、目录项、页码或等价定位。

来源清单负责“材料从哪里来、能不能读”；结构抽取负责“这一段材料在哪里、能支撑什么”。两者分开，配方师和创作者才能回到同一块原料。

## 2. 先保来源，再划定材料单元

先保持每份 Markdown 来源完整，再划定后续同事能回到的材料单元。材料单元是理解颗粒，不是派工切块；它让判断能回到哪份来源、哪个章节、哪几行。

划分顺序很简单：

1. **先看天然结构**：书籍和电子书优先找目录，核对目录项与正文里的章名、页码或标题是否对应。目录可靠，就按目录的章、节，再结合自然论证段划分。
2. **再看稳定结构**：论文、报告按章节、标题层级、论证段或数据段划；笔记和文稿按主题段、标题或自然话题划。
3. **最后看语义边界**：扫描 PDF、OCR 文本、格式异常的电子书或其他稀奇文档，如果没有可靠目录和稳定标题，就按问题转换、论证推进、概念更替、案例进入和收束处划分。不要让 OCR 误识别的小标题、页眉页脚或样式碎片替材料定结构。
4. **多源先单源后跨源**：多源文档包先理解每个单源，再整理跨源材料单元。

无论按哪一种方式划分，都要在材料单元表里写清划分依据。比如“目录第 2 章，对应正文 L320-L890”，或“无可用目录，按论证从问题提出到机制解释划为一单元”。

划定材料单元不是为了把材料切碎，而是为了留下可追溯的工作颗粒。每个单元都要有 Markdown 起止行或等价位置。

只要 `sources/markdown/` 里不止一个 Markdown 文件，位置参数就不能只写行号。默认写成 `文件名:起止行`；有页码、目录项或标题层级时，也先写文件名，再写等价位置。

## 3. 再读每个单元的劲道

逐个读取材料单元，读一个写一个。不要全部读完后凭印象写总论。

每个单元至少写清：

- **核心命题**：这部分真正回答什么问题。
- **价值**：它提供了哪一层新理解。
- **颠覆**：它修正了什么常见看法。
- **支撑**：关键证据、故事、案例、研究、经验或论证。
- **边界**：它没有证明什么，不能说满什么。
- **后续用途**：更适合做入口、建架、推进、反例、行动，还是边界提醒。

这里的语言要克制。材料先保持自己的立场，后面的配方再决定怎样面向顾客。

## 4. 折叠出整体结构

单元读完后，再写整体判断：

- **终极核心命题**：全材料反复折叠、不断回到的意义源头。
- **论证地图**：围绕核心命题的回应，从起点到终点，材料怎样递进、转折、反驳、补充或收束，去回答核心命题。
- **KeyN / 关键命题**：真正反转俗见、能撑起后续配方和创作的关键判断。

多源材料要特别小心。必须区分：

- **原生结构**：材料本身明确呈现的结构，需要尊重但非严格遵循。
- **策展连接**：主理人或研究同事在多源之间整理出的连接，为结构判断的主体。
- **处理矛盾**：在策展时，不把多源材料强行统一，尊重材料之间的矛盾关系。

策展连接要以策展连接的身份出现：它既不是原文结论，也不是材料之间的流水记账。

多源处理的目的，是在保留差异和矛盾的前提下，判断哪些连接能成立，哪些只能暂放。

## 5. 输出

单人验料时，洞见发掘者写入 `workspace/structure-extraction.md`，并另写 `workspace/creation-soul.md`。结构抽取负责材料事实，creation-soul 负责创作面包师的存在底色；两者不能混写。

多人验料时，局部同事先写自己的 `workspace/extraction-*.md` 中间文件；最终 `workspace/structure-extraction.md` 和 `workspace/creation-soul.md` 只在合并环节生成一次。

```markdown
# 结构抽取

## 来源清单

## 材料单元表
| 单元 | 位置参数（文件名:起止行 / 目录项 / 页码） | 划分依据 | 核心命题 | 价值 | 颠覆 | 支撑 | 边界 | 后续用途 |
|---|---|---|---|---|---|---|---|---|

## 材料单元命题

## 终极核心命题

## 论证地图

## KeyN / 关键命题

## 多源材料关系（可选）

## 风险与边界

## 给配方师的提醒
```

`workspace/creation-soul.md` 的生成规则不在这里重复展开；按 `references/creator-soul-and-direction.md` 写。这里仅守三条边界：第一人称，人物故事，不能把艺术加工写成作者事实或隐私推断。多人验料时，soul 必须从合并后的全材料气质里生出来，不能由某个局部同事提前替整炉定调。

## 6. 多同事协作

结构抽取默认一个洞见发掘者完成。多同事协作只在容量确实超过单个同事可稳定处理的范围时启用：`sources/markdown/` 总正文超过约 18 万中文字符，或 Markdown 文件数超过 10 个。低于这个范围还分给多个同事，通常是在用并行制造合并成本。

多人验料的协作关系很简单：局部同事负责把自己范围内的材料读透，合并者负责把局部判断放回同一炉里。局部同事不是缩小版的全书洞见发掘者，不替整份材料写终极核心命题、全局论证地图、全局 KeyN，也不召唤创作者 soul。

分工优先按完整来源文件或清楚的来源组，而不是按任意字数划一刀。如果单一来源大到必须分段协作，只能按原书目录、章节、标题层级或自然主题边界划分；中间文件名要带来源名，例如：

- `workspace/extraction-human-condition-ch01-03.md`
- `workspace/extraction-notes-20260515-part1.md`
- `workspace/extraction-reports-a-to-c.md`

每位局部同事只写自己的中间文件。中间文件可以沿用结构抽取模板，但标题要改成局部范围，例如 `# 局部验料：Human Condition ch01-03`，并只写这几项：

- 分配范围：来源文件、目录项、章节、标题层级、页码范围或主题边界。
- 材料单元表：每个单元都带 `文件名:起止行` 或等价位置。
- 局部命题：这部分材料真正支撑什么。
- 支撑与边界：关键证据、论证、故事、反例，以及不能说满的地方。
- 给合并者的提醒：哪些判断有全局潜力，哪些只适合留在局部，哪些需要回原文复查。

全部返回后，主理人亲自读取、合并成唯一的 `workspace/structure-extraction.md`，并检查材料单元是否完整、位置参数是否可追、关键命题是否重复或冲突、哪些连接属于原文结构、哪些属于策展连接。材料仍然过大时，可以只委派一位合并型洞见者；合并型同事接收所有中间文件和必要原文入口，负责形成唯一最终结构，不再让多个同事各写一份全局方案。

合并时先查位置参数。只写 `L120-L180`、没有文件名的多文件验料结果，不进入配方；先退回补 `文件名:起止行`。
