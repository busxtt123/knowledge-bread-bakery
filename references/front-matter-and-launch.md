# 前置内容与结课地图

## 0. 这页是什么

本页只处理一件事：正课写完后，怎样补上作品入口，并在最后一讲把整张知识地图摊开。

它不是逐讲创作 SOP，也不是课程详情页写作手册。它解决的是入口和收束：用户为什么愿意开始，开始前应该先知道什么，走完整段路后怎样看见整体结构。

## 1. 为什么最后写

前置内容放在正文前面，但不在生产最开始写。知识面包先完成材料结构、课程规划和正课正文，再回头写入口。

这样做有两个理由：

- 只有正课真正写完，创作者才知道这张地图的真实路线、惊奇点和终点，不会用先验模板冒充入口。
- 发刊词、前言和导论负责把用户带进已经成形的作品，而不是提前规定作品必须长成什么样。

所以流程是：配方阶段只规划前置内容类型和功能；最后一讲完成后，创作者把结课知识地图写出来，再回到 `workspace/manuscript.md` 前部补写前置内容。

## 2. 选哪种入口

无论单篇还是课程，都要有入口。入口类型按内容形态选：

- **单篇长文**：写“前言”。它不做强销售，只说明为什么读、怎样读、读完能带走什么。
- **多讲课程**：写“发刊词 + 导论”。发刊词是第 0 讲，负责启动动机；导论是进入正式学习前的框架，负责地图。

判断多讲课程的标准不是字数，而是用户是否需要一段连续学习旅程：多讲、多阶段、多次认知转折，或者需要轻量练习帮助吸收。

## 3. 发刊词

发刊词是第 0 讲。它不是正式教学内容，而是课程说明书兼动机启动器。它面向还没决定要不要开始的人，回答三件事：

1. 为什么要学这门课。
2. 为什么是现在。
3. 为什么跟这条路线 / 这位讲述者学。

发刊词的气质偏感性。它把课程的原问题翻译成用户能感到的现实张力：旧模型的解释力在哪里停住，这份材料为什么能给我们一张更有解释力的新地图。

可用结构：

- **张力开场**：从现实机制、旧模型、共同经验或材料反差开场，让人知道问题为什么值得进入。
- **旧模型的成立与边界**：承认熟悉办法为什么有道理，再说明它在哪些结构里解释力不够。
- **课程承诺**：这门课会带用户走过怎样的地图，最终多出什么分辨力。
- **可信理由**：为什么这份材料、这条路线、这位讲述者值得跟随。没有真实老师资料时，不编头衔、不编经历，只说材料本身和本次策展的可信处。
- **感性收尾**：邀请用户开始，不把答案提前讲透。

发刊词可以使用故事和金句；承诺、权威和讲述者身份都要来自真实材料和本次策展，soul 只留下判断气息，不泄漏成真实作者事实。

维护读者认知体验底线：发刊词用旧模型、现实机制、结构错位或解释框架不足启动动机。需要写人的局限时，用“我们”或“人在这种结构里”这类共同经验，让读者从容走进问题。

## 4. 导论

导论是正式学习的入口。它跟在发刊词之后，放在第一讲之前。

导论的气质偏理性，任务是建立学习框架：这门课研究什么，不研究什么；整张认知地图大致怎样展开；用户应该用什么方式进入后面的讲次。

导论不是发刊词的理性版，也不是第一讲的摘要。它做三件事：

- **界定边界**：这门课要解决的问题、不会解决的问题、不能说满的地方。
- **建立地图**：出发地、关键地标、路线和终点，让用户知道后面不是材料堆叠。
- **说明学法**：用户每讲应该关注什么，是看案例、看模型、看反差，还是做一次轻量练习。

知识面包里，导论也在正课写完后补写。它放在前面，是阅读顺序；它最后写，是生产顺序。

## 5. 前言

前言用于单篇长文。它比发刊词轻，也不需要另写导论。

前言回答三个问题：

- 这份内容为什么值得现在读。
- 读的时候应该抓住哪条主线。
- 读完能带走什么看法、语言、分辨力或行动方向。

前言像给顾客递上第一口：告诉他这块面包从哪里尝，为什么值得尝，尝完大概会明白什么。它轻，不抢正文的风头。

前言同样要维护读者认知体验底线。它可以指出旧理解的局限，但阅读动力来自材料和现实问题本身。

## 6. 最后一讲摊开知识地图

最后一讲有一个额外任务：正课内容完成后，把整张知识地图摊开讲一遍。

这不是普通总结，也不是目录回放。它要让前面所有游历过的地标重新组织起来，让用户突然看见：原来这些材料共同构成了一张新地图，我对这个问题的理解已经换了位置。

合格的结课地图有四个特征：

- **在正课完成后出现**：先完成最后一讲自己的材料任务，再摊开地图。
- **组织关系，不复述内容**：重点讲地标之间的关系、路线的必然性和终点的变化。
- **力量来自整体编排**：用户不会觉得这是硬插总结，而会觉得前面一路走到这里，地图自然该出现。
- **完成认知重构**：用户能说清自己原来怎样看，现在怎样看，为什么这张新地图更有解释力。

最后一讲写完结课地图后，创作者再回到 `workspace/manuscript.md` 前部补写前言，或发刊词 + 导论。

## 7. 写入规则

前置内容写入 `workspace/manuscript.md` 的开头，不另开孤立文件。写入时只做增量补丁，不覆盖已有正文。

标题层级要守住交付边界：一级标题只给整本产品标题；发刊词、导论和前言都是正文入口，用二级标题。导论内部的小节用三级标题。这样 PDF 才会把它们当正文目录项，而不是误抽成封面标题或一组并列大章。

整理 `delivery/final.md` 时不要把这些标题降成三级。`### 前言`、`### 发刊词` 或 `### 导论` 会让入口从目录层级里掉下去；标准 PDF 脚本会直接报出行号，先改回二级标题再生成。

推荐顺序：

- 单篇长文：`## 前言` → 正文。
- 多讲课程：`## 发刊词` → `## 导论` → `## 第 1 讲`。

写完后，在 `workspace/project-log.md` 告诉主理人：补了哪种入口，为什么这样选，结课知识地图写在哪里，是否有无法确认的承诺或边界。

## 8. 走偏气味

- 一开始就写发刊词或导论，并反过来规定正课必须长成某种形态。
- 把 `# 发刊词`、`# 导论` 或 `# 前言` 写成一级标题，导致交付脚本把正文入口误当成整本产品标题。
- 把 `## 发刊词`、`## 导论` 或 `## 前言` 改成三级标题，导致 PDF 目录和分页层级错误。
- 单篇长文没有任何前言，用户直接被扔进正文。
- 多讲课程只有发刊词，没有导论；或者只有导论，没有动机入口。
- 发刊词像正式教学，急着讲知识。
- 导论像销售文案，只渲染价值，不建立地图和边界。
- 发刊词或前言把阅读动力放在指责、焦虑或亏欠上，旧模型、现实机制、结构错位或材料反差没有承担张力。
- 结课地图只复述每讲内容，没有组织整体关系。
- 前置内容夸大承诺、编造老师经历、泄漏 soul 设定，或制造虚假权威。
