# Subagent 故障与接管

## 0. 接管位置

本页只处理 subagent 异常：工具不可用、无可用 provider、超时、部分写入、写错位置、覆盖风险和文件系统隔离。正常委派看 `SKILL.md` 和各工位 SOP。

故障处理顺序是：先保护现场，再判断剩余工作，最后决定主理人接管还是缩小范围后重派。

## 1. 故障信号

- 工具不可用：当前环境没有 subagent / worker / delegate 工具。
- 无可用 provider：返回 `No available providers`、`HTTP 503` 或类似信息。
- 超时：长时间无响应，或任务被系统中断。
- 部分写入：subagent 没完整返回，但工作文件已有新增内容。
- 写错位置：写到孤立文件，没有续写到指定正文。
- 覆盖风险：文件大小或行数异常变小，可能覆盖了前文。
- 文件系统隔离：主会话能读到文件，subagent 却报告文件不存在或搜索不到。

## 2. 第一反应

不要立刻按原派工说明重派。先做四件事：

1. 读取 `workspace/manuscript.md`、`workspace/project-log.md` 和必要的 `workspace/creation-calendar.md`。
2. 判断当前讲或当前讲组是否已有部分正文、创作日记、给主理人的沟通或覆盖信号。
3. 在 `workspace/project-log.md` 记录故障信号、发生时间、返回信息、已完成什么、缺什么。

## 3. 接管或重派

直接接管的情况：

- 当前环境不支持委派。
- 同一任务连续委派失败超过 3 次。

接管后仍按工位 SOP 走；主理人亲自写，也不能跳过现场判断和出站裁决。

如果任务失败不超过3次，那么继续正常委派。

## 4. 部分写入处理

部分写入不是自动失败，也不是自动通过。主理人检查：

- 新增内容是否接在正确位置。
- 是否只写派工范围内的当前讲或当前讲组。
- 是否有规划修订告知或不确定处写入 `project-log.md`。
- 是否覆盖前文、创作日记或主理人工作记录。
- 是否符合当前讲或当前讲组的交付点和材料取舍。

能用的内容保留，不能用的内容删除或改写。处理过程写入 `project-log.md`。

## 5. 覆盖风险

看到以下信号，立即停止后续委派：

- `manuscript.md` 行数或大小突然大幅变小。
- 前面讲次标题消失。
- 文件只剩当前讲内容。
- `creation-calendar.md` 创作状态异常丢失。
- `project-log.md` 丢失规划修订告知、出站裁决或故障记录。

按 `references/recovery/recovery-from-manuscript-overwrite.md` 恢复。恢复前不要启动下一讲或下一组，不要整理交付稿。
