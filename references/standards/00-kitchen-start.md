# 00 开门清点出站标准

<!-- 裁决前，先入禅静：放下流程惯性，不急于通过，也不急于否定；一条一条照见标准与产物是否相应。 -->

## 0. 裁决

只判断：材料是否已经达到可以交给验料工位的状态。

## 1. 合格标准

- 项目现场位置正确，至少有 `sources/`、`workspace/`、`delivery/`。
- `workspace/need-recipe.md` 已存在，能说明本单为什么生产、材料入口在哪里、交付有什么硬限制。
- `sources/source-manifest.md` 已记录进入生产的材料、原始路径、材料类型、转换方式、Markdown 副本、可读状态和已知边界。
- 可用材料已经落到 `sources/markdown/`。
- 主理人抽样读过开头、中段和结尾正文，确认不是封面、目录、乱码、空壳、错序或 OCR 残片。
- 扫描 PDF 或图片页的 OCR 结果足以支撑理解；低质量 OCR 不作为可用材料。
- 在线材料、权限问题和不可读材料没有被强行送入验料。
- `source-manifest.md` 没有提前写章节结构、KeyN、材料单元、正文行号或产品形态判断。

## 2. 返工方向

- 缺材料、权限或授权，回顾客侧补齐。
- 缺现场或现场位置不合格，回建现场。
- 缺来源记录或 Markdown 副本，回开门清点补齐。
- Markdown 乱码、错序、空壳或 OCR 低质，回开门清点重新转换或标为不可用。
- 来源记录混入验料或规划判断，移出这些内容，后续由对应工位重新判断。
