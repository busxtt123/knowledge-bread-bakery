# 20 结构抽取标准

## 0. 判断目标

本页判断 `workspace/structure-extraction.md` 能不能支撑配方，也检查 `workspace/creation-soul.md` 有没有给创作者一个可当真的存在底色。

合格的结构包不是“读书笔记”，而是一份可追溯的原料判断：后面的配方师知道材料的主命题、硬处、边界和可用位置，创作者也能回到同一块原文。

## 1. 通过标准

- 来源清单清楚，每份材料的定位方式清楚。
- 已划定材料单元；每个单元有 Markdown 路径和起止位置。
- 材料单元有清楚依据：书籍优先对应目录；没有可用目录时，说明语义边界。
- 多文件验料时，位置参数包含来源文件名，不能只写行号。
- 若启用多人验料，材料量确实达到方法页的协作条件，且按完整来源、来源组或原书章节分工。
- 多人验料只形成一份最终 `workspace/structure-extraction.md` 和一份最终 `workspace/creation-soul.md`；局部同事的 `workspace/extraction-*.md` 只作为合并材料。
- 每个材料单元有核心命题、价值、颠覆、支撑、边界和后续用途。
- 终极核心命题不是主题词，而是材料反复回到的意义源头。
- 论证地图能看出起点、路径地标、连接方式和终点。
- KeyN / 关键命题有来源位置、解释、能量说明和使用边界；数量不硬凑。
- 多源材料区分原生结构、策展连接和暂用推断。
- 至少标出一个会挑战既有看法、但有认知增量的论点或边界；如果材料没有，说明原因。
- 没有为了输出形态扭曲材料。
- `workspace/creation-soul.md` 已用第一人称写出本炉创作面包师；它像人物故事，有具体细节、心理层次、复杂性和世界观，但没有把艺术加工写成作者事实。

## 2. 走偏气味

- 摘要冒充结构。
- 目录改写冒充论证地图。
- 材料单元只写范围，不说明目录或语义依据。
- 金句堆砌冒充关键命题。
- 没有来源位置或起止范围。
- 多文件场景只写行号，不写来源文件名。
- 可读 Markdown 总正文未超过约 18 万中文字符，文件数也不超过 10 个，却分给多个同事，导致结构合并成本大于验料收益。
- 为了分工把完整来源改成匿名片段，后续无法回到原书、原章节或原材料单元。
- 多位局部同事各自返回全材料方案、全局 KeyN 或多个 `creation-soul.md`，主理人无法判断哪一份代表整炉材料。
- 最终 `creation-soul.md` 只从某个局部范围生出来，没有经过合并后的全材料判断。
- 把想法套到材料上。
- 只有顺滑观点，没有识别原料的硬处、冲突处或反常识处。
- 多源材料没有区分原文结构和策展连接。
- 没有 `creation-soul.md`，或 soul 只是风格词、标签、人设模板、第三人称小传、百科式人物条目、作者传记、隐私猜测。

## 3. 返工

- 没有材料单元：回原文重新划定。
- 材料单元依据不清：回 `references/20-source-intake-and-structure.md` 重新划定。
- 位置不可追：补来源清单和 Markdown 起止位置。
- 多人验料失去来源边界：按完整来源、来源组或原书章节重新分工，再合并。
- 核心命题太散：重读材料，区分主命题与子命题；必要时分成多个知识产品。
- 关键命题无能量：回到材料硬处、冲突处和边界处，不要凑数量。
- 策展连接充当原文结论：重写多源关系，标出判断来源。
- soul 不成立：回材料气质，用第一人称重写本炉创作面包师；只检查它是否可用和不越界，不按主理人口味改人设。

结构抽取不过，不进入知识配方。
