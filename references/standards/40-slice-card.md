# 40 当前切片派工标准

## 0. 目标

本页只判断当前切片派工说明是否能把创作者带进现场。

好的派工说明让创作者接得动：知道自己是谁，前文到哪里，当前这块在整炉里起什么作用，原文该读哪一段，哪些材料该留在背景里，哪些沟通要写给主理人。它给火候，不给模板。

## 1. 通过标准

- 当前切片在 `workspace/course-plan.md` 中的位置清楚。
- 本切片在整炉里的作用清楚。
- 入口判断清楚：顾客为什么愿意从这里进入。
- 材料最有劲的地方清楚。
- 要形成的新组织 / 新排列清楚。
- 火候 / 分量判断清楚。
- 读者认知体验底线清楚：如果本讲需要呈现冲突或失效，张力来自材料、旧模型、现实机制或结构错位；入口能让读者从容进入材料。
- 若当前切片是最后一讲，派工说明明确写清两项额外任务：正课内容完成后摊开结课知识地图；再按 `workspace/course-plan.md` 的前置内容规划，把单篇长文的前言或多讲课程的发刊词 + 导论补到 `workspace/manuscript.md` 前部。
- 若当前切片不是最后一讲，派工说明不要求创作者提前写前言、发刊词或导论。
- 主料清楚；如有多个材料，派工说明说清它们如何共同拧成一股劲。
- 原文材料包有来源文件、阅读目的和目录项 / 页码 / 标题层级 / Markdown 起止行；多文件场景用 `文件名:起止行`。
- 可选、延后、删除材料边界清楚。
- 当前讲派工说明明确交给创作者 `references/agents/40-creation-baker.md` 作为稳定工位 SOP。
- 派工说明已贴出 `workspace/creation-soul.md` 全文，而不是只给路径或把 soul 改写成风格说明。
- 正文 `workspace/manuscript.md`、创作日记 `workspace/creation-calendar.md` 可读；`workspace/project-log.md` 存在或派工说明要求创作者按需创建。
- 派工对象清楚：默认委派当前讲 subagent；若 subagent 不可用，派工说明记录主理人接管理由。
- 任务边界清楚：当前讲独立启动、独立品尝；下一讲等前文真正落地后再接上。
- 派工说明只补本讲差异，没有重复粘贴一套会和工位页冲突的通用流程。
- 回炉路线清楚：创作者写完当前讲后有禅定时刻；回炉请求可以来自 soul 的不舒服、方向上的作品追求、材料现场里“这样写更对”的判断，或读者认知体验底线；请求写入 `workspace/project-log.md`，课程规划由主理人或配方站修订。
- 文件安全清楚：已有 `manuscript.md`、`creation-calendar.md` 和 `project-log.md` 以增量方式接续。

## 2. 走偏气味

- 只说“写第几讲”，没有说明这一块在整炉里的作用。
- 给了原始材料全集，让创作者自己决定取舍。
- 要求多个故事都展开。
- 没有入口判断，只给概念入口。
- 没有火候 / 分量判断。
- 入口把阅读动力放在指责、羞耻或亏欠上，材料和现实机制没有承担张力。
- 没有停顿回看的安排。
- 最后一讲派工说明没有要求结课地图和前置内容补写。
- 让创作者在第一讲或中间讲提前写发刊词、导论或前言。
- 没有明确委派当前讲 subagent，也没有记录 subagent 不可用的接管理由。
- 把相邻讲次并行派出，切断了创作连续性。
- 没有贴出本炉 soul 全文，只给路径或风格说明；或把创作日记当成主理人验收表。
- 没有给 `project-log.md` 主理人工作记录入口，导致来自 soul、方向、材料现场或读者认知体验底线的回炉请求、规划修改建议、不确定处和品尝判断只能混进创作日记或散落在会话里。
- 只有“写得有对象感 / 有营养”，没有具体材料、火候和切片作用。

## 3. 返工

派工说明接不住创作者时，回配方补本切片在整炉里的作用、入口判断、材料取舍、火候分量、读者认知体验底线、原文材料包、soul 全文交付、`project-log.md` 主理人工作记录入口、禅定回炉路线和当前讲任务边界；最后一讲还要补结课地图和前置内容补写要求。材料位置不清时回结构抽取。
