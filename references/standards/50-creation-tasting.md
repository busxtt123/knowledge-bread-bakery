# 50 创作品尝标准

## 0. 目标

品尝只判断一件事：当前切片是否写到位。主理人读正文、`workspace/project-log.md` 和必要原文；创作日记服务创作者连续性，不作为通过或不通过的依据。

写到位的标志是：这一块在整炉里起到了它该起的作用；顾客愿意入口，嚼起来能尝到材料的劲，吃完以后有了一个更清楚的看法。完整、故事和漂亮句子都要服务这个结果。

## 1. 单切片品尝

### 1.1 这一块是否有火候

- 前文续到位了：读者能感觉这一讲从上一讲自然推出来。
- 本切片作用清楚：它知道自己在整炉里负责第一口、揉劲、发酵、定型，还是收束。
- 入口成立：第一段给出愿意继续尝的一口。
- 材料有劲：事实、场景、人物、机制或边界能托住判断。
- 篇幅刚好：该展开处展开，该停处停。

### 1.2 认知是否发起来

- 顾客带来的看法有回应：读者能认出自己为什么会这样想。
- 新排列从材料里推出来：读者能跟着材料看见位置如何移动。
- 具体材料让抽象判断变得嚼得到、分得清、记得住。
- 关键处有“原来还能这样排”的一瞬。
- 出炉时留下更清楚的看法、语言、分辨力、地图或行动方向。

### 1.3 讲述姿态是否稳

- 顾客站在案台前看材料：他以一起看火候的人出现。
- 已有经验成为发酵底子：正文能说明它为什么成立，也能带它往前走一步。
- 事实、场景、机制和边界承担重量：判断从材料中生出。
- 新模型像重新整理案台：同一批材料换了位置，解释力变强。
- 语气有火候：能带人看清楚，也留得住分寸。
- 背后有一个具体创作者在判断：正文不暴露 soul 设定，但能感到不可替代的取舍、敏感点和分寸。
- 维护读者认知体验底线：若正文呈现冲突、失效或不安，张力来自材料、旧模型、现实机制、结构错位或解释框架不足；读者能从容进入材料。

### 1.4 方向是否拉开

- 新锐：没有停在俗知俗见或常见讲法上，至少有一个更锋利、更外部或更有解释力的角度。
- 性感：知识有势能，读者能感到“这件事为什么非知道不可”，而不是在读说明书。
- 个性鲜明：换一个普通创作者不容易写成同样效果；语言、取舍和判断有本炉 soul 的底色。
- 科学依据稳：方向再大胆，事实、来源、边界和推断仍然清楚。

### 1.5 连续性是否清楚

- 续上前文已经建立的概念、情绪或问题。
- 给后文留下明确可接的线。
- 若有回炉请求、规划修改建议、偏离规范或不确定处，`workspace/project-log.md` 已说明理由和需要主理人判断的地方。
- 启动顺序清楚：上一讲已品尝通过；若中途接管或返工，主理人有说明。
- 没有并行写相邻讲次：当前讲启动时，上一讲正文和 `project-log.md` 里的主理人品尝判断已经落地。

### 1.6 如果这是最后一讲

最后一讲还要检查入口和收束是否闭合：

- 正课内容先完成，不用结课地图替代最后一讲自己的材料任务。
- 结课知识地图已经在最后一讲末尾摊开，重点组织整体关系，不复述每讲目录。
- 结课地图能让用户看见：原来一路走过的地标共同构成了一张新地图。
- `workspace/manuscript.md` 前部已经补写正确入口：单篇长文有前言；多讲课程有发刊词 + 导论。
- 前置内容标题层级正确：前言、发刊词或导论用二级标题，导论内部小节用三级标题，不占用整本产品标题层级。
- 发刊词、导论或前言没有夸大承诺、编造老师经历、泄漏 soul 设定或制造虚假权威。
- `workspace/project-log.md` 说明了入口类型、选择理由、结课地图位置和需要主理人裁断的边界。

### 1.7 回炉请求是否成立

创作者提出回炉请求时，先守边界，再看作品是否因此更好。回炉请求可能来自四处：soul 上的不舒服，方向上追求好作品、避免标准工业品的选择，材料现场里“这样写更对”的判断，或正文把阅读动力放在指责、羞耻、亏欠上。

可以接受的回炉请求，要同时满足四件事：

- 事实、来源、材料位置和推断边界没有被破坏。
- 分量和交付形态仍然可控。
- 改法让观点更准确、逻辑更清楚、主料更有力，或让本讲的新锐、性感、个性更明显。
- `workspace/project-log.md` 说清了哪里不对、为什么更对、建议回到哪里改，以及改完怎样继续服务顾客和整炉走向。

回炉请求没有成立时，通常是材料证据不足、脱离整炉路线，或把方向追求变成任性偏离。主理人在 `workspace/project-log.md` 写清判断：成立就说明返当前讲、派工说明、配方、验料还是需求；没有成立就写清原因，再返当前讲继续修。

## 2. 整体品尝

全部切片完成后，再看整体：

- 整炉有清楚走向，多个章节共同烤成一只面包。
- 每个切片各有火候，彼此不重复抢味。
- 顾客一路被带着吃下去，阅读动力来自入口、材料和节奏。
- 关键概念、主料、对照、边界和停顿有节奏地出现。
- 整体有变化的呼吸：每讲根据材料和作用形成自己的入口、转折和结尾。
- 最后一讲摊开了完整知识地图，整体结构的力量能把前文重新组织起来，而不是只做目录式回顾。
- 前置内容已经放在正文前部，类型和作品形态匹配。
- soul 没有变成设定表演，但整炉能感到一个活生生的创作者在场。
- 最终交付适合顾客实际阅读和使用。

## 3. 返工路线

- 入口硬：回配方和创作，重新找第一口。
- 材料没出味：回原文，重选主料或改写主料呈现。
- 创作者指出更好的材料但未说明：回 `project-log.md`，补清原材料位置、替代理由、对当前讲任务和前后文的影响。
- 像在填表：回创作，重新判断这一块的作用、主料和余味。
- 合格但无魂：回创作，重新承接派工说明中的 soul；如果派工说明没有贴出全文，主理人先补齐，再判断是否需要在 `project-log.md` 写回炉请求并修订当前讲规划。
- 方向平：回创作，检查新锐、性感、个性鲜明哪一项没有成立。
- 故事过载：回配方，重做材料取舍。
- 概念夹生：补具体材料、对照或边界。
- 语气带偏：回到顾客的初始看法怎样成立，再让材料完成牵引。
- 读者难以从容进入材料：回创作，重写入口和张力的来处，让阅读动力回到材料、旧模型、现实机制或结构错位。
- 有回炉请求但没说明：回 `project-log.md`，补清请求来源、建议返站、规划修改建议、偏离规范或不确定处。
- 最后一讲没有结课地图：回最后一讲，正课内容完成后补写整体知识地图。
- 前置内容缺失或类型不对：回最后一讲后的前置补写；单篇长文补前言，多讲课程补发刊词 + 导论。
- 发刊词像正式教学、导论像销售文案：按 `references/front-matter-and-launch.md` 重写对应入口。
- 没有逐讲品尝：补读当前讲正文、`project-log.md` 和必要原文，把品尝通过 / 不通过判断写入 `project-log.md` 后再决定是否进入下一讲。
- 发现两讲并行写成：先停止后续创作，主理人按真实完成顺序重读 `manuscript.md` 和 `project-log.md`，让创作者自行接回创作状态；不能续上前文的讲次退回当前讲重写。创作日记不作为通过依据。
