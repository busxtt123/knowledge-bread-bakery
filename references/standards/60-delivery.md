# 60 交付出站标准

<!-- 裁决前，先入禅静：放下流程惯性，不急于通过，也不急于否定；一条一条照见标准与产物是否相应。 -->

## 0. 裁决

只判断：成品是否已经达到可以交给顾客的状态。

## 1. 合格标准

- 默认交付按成品标题命名的 PDF；顾客明确要求 Markdown 时，才同时交付 `delivery/final.md`。
- `delivery/final.md` 来自 `workspace/manuscript.md`，只做标题层级、衔接、重复、明显语病和交付结构统一；没有拼入过程文件。
- `delivery/final.md` 只有一个一级标题，一级标题是整本产品标题；一级标题下一行是给顾客看的 `> 副标题`。
- 前言、发刊词、导论和各讲正文使用二级标题；导论内部小节和讲内小标题使用三级标题或正文加粗。
- 多讲课程有发刊词和导论；单篇长文有前言。
- 结课地图不作为读者可见生产名进入目录。
- PDF 只用标准脚本生成：`python scripts/build-mobile-pdf.py delivery/final.md "delivery/标题名称-16讲.pdf"`。
- 标准脚本结构预检通过，目录预览像读者会翻阅的书或课程骨架。
- PDF 文件存在且非空；`pdfinfo` 能读取页数和尺寸；`pdftotext` 能抽取正文且没有明显乱码、本地路径、`file://`、调试信息或过程标记。
- 手机视角抽查看过封面、目录、普通正文页、列表页，以及引用 / 表格 / 代码块页面。
- 过程现场不默认交付。
- 高风险主题已经完成边界复查，并应用到正文或交付稿。

## 2. 返工方向

- 标题、副标题、目录和前置内容问题，回 `delivery/final.md` 或最后一讲补写。
- PDF 生成失败，先修标准脚本或运行环境，不分叉新脚本。
- PDF 可生成但不可读，回整理稿拆长表格、减密集列表；样式问题回脚本。
- 过程痕迹外露，回整理稿清理。
- 交付范围过宽，收回过程文件或另行整理。
- 高风险边界未复查，先请边界检查员复查并修正文或交付稿。
