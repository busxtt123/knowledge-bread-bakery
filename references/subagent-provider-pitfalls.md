# Subagent 故障与接管

## 0. 这页是什么

这页只处理 subagent 异常：工具不可用、provider 不稳定、超时、部分写入、写错位置、覆盖风险和文件系统隔离。正常委派已经写在主流程和各工位 SOP 里，本页不重复。

它存在的理由是保护现场。逐讲创作的真实进度可能已经写入 `workspace/manuscript.md`、`workspace/creation-calendar.md` 或 `workspace/project-log.md`；如果主理人没检查现场就重派，同一讲可能被覆盖、重复、写散，或者再次超时。

故障处理按一个顺序走：先保护现场，再判断剩余工作，最后决定主理人接管还是缩小范围后重派。

## 1. 故障信号

- **工具不可用**：当前环境没有 subagent / worker / delegate 工具。
- **无可用 provider**：返回 `No available providers`、`HTTP 503` 或类似信息。
- **超时**：长时间无响应，或任务被系统中断。
- **部分写入**：subagent 没完整返回，但工作文件已有新增内容。
- **写错位置**：写到孤立文件，没有续写到指定正文。
- **覆盖风险**：文件大小或行数异常变小，可能覆盖了前文。
- **文件系统隔离**：主会话能读到文件，subagent 却报告文件不存在或搜索不到。

## 2. 第一反应

不要立刻按原派工说明重派。先做四件事：

1. 读取 `workspace/manuscript.md` 和 `workspace/project-log.md`；检查 `workspace/creation-calendar.md` 是否有新增、丢失或覆盖信号。创作日记只用于保护连续现场，不作为品尝依据。
2. 判断当前讲是否已有部分正文、创作日记、给主理人的沟通或异常覆盖。
3. 在 `workspace/project-log.md` 记录故障信号：发生时间、返回信息、已完成什么、缺什么。
4. 判断派工说明是否过重：是否一次要求读太多原文、写太长正文、同时处理多个转折。

## 3. 接管或重派

### 3.1 直接接管

满足任一情况，主理人直接接管当前讲：

- 只剩少量正文、收束、创作日记或主理人沟通。
- 当前讲派工说明已经清楚，重派只会浪费时间。
- subagent 连续不可用或 provider 不稳定。
- 已经出现部分写入，需要人工判断能不能保留。
- 主会话确认文件存在，但 subagent 报告文件不存在，属于文件系统隔离。

接管后仍按 `references/agents/40-creation-baker.md` 走：承接 soul、读前文、创作日记、规划和原文，再补正文；创作想法写 `creation-calendar.md`，故障和沟通写 `project-log.md`，再品尝。主理人亲自写，也不能跳过现场判断和品尝。

### 3.2 缩小后重派

只有剩余工作仍然清楚、且 subagent 环境恢复时，才重派。重派要缩小范围：

- 只写当前讲的一段，不写整讲。
- 只处理一个主锚点，不重新读全材料。
- 只补停顿回看、收束或出炉后留下什么。
- 只做边界复查，不改正文。
- 原材料过大时，主理人先把当前讲必需片段提取到项目现场，再让 subagent 只读片段文件。

重派说明写清从哪一行或哪一节之后续写，并说明已有内容如何保留。

## 4. 部分写入怎么处理

部分写入不是自动失败，也不是自动通过。主理人检查：

- 新增内容是否接在正确位置。
- 正文是否只写当前讲。
- 是否有回炉请求、规划修改建议或不确定处写入 `project-log.md`。
- 是否覆盖前文、创作日记或主理人工作记录。
- 是否符合当前讲唯一交付点和材料取舍。

能用的内容保留，不能用的内容用补丁删除或改写。处理过程写入 `project-log.md`，让后续同事知道哪些是保留、哪些是返工。

## 5. 覆盖风险

看到以下信号，立即停止后续委派：

- `manuscript.md` 行数或大小突然大幅变小。
- 前面讲次标题消失。
- 文件只剩当前讲内容。
- `creation-calendar.md` 创作状态异常丢失。
- `project-log.md` 丢失回炉请求、规划修改建议、品尝判断或故障记录。

按 `references/recovery-from-manuscript-overwrite.md` 恢复。恢复前不要启动下一讲，不要整理交付稿。

## 6. 快速判断

| 现象 | 判断 | 动作 |
|---|---|---|
| 工具不存在 | 环境不支持委派 | 主理人接管，并记录原因 |
| `No available providers` / `HTTP 503` | subagent 暂不可用 | 不按原派工说明重试，主理人接管 |
| 超时但文件有新增 | 可能部分完成 | 先读文件，再决定保留 / 补写 / 返工 |
| 超时且无新增 | 任务可能过重或环境不稳 | 缩小范围或主理人接管 |
| 超时且原材料过大 | 读取成本过高 | 先提取当前讲片段，再缩小重派 |
| 写到孤立文件 | 没续上正文 | 读孤立文件，可用部分补回正文 |
| 文件变小 | 可能覆盖 | 停止委派，进入恢复流程 |
| 文件系统隔离 | subagent 无法访问主会话文件 | 主理人直接接管 |
