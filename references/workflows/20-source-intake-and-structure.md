# 20 材料结构抽取方法

## 0. 验料位置

本页是洞见发掘者在验料站使用的方法页，解决一件事：怎样把可读材料划成可追溯的材料单元，并从单元里读出材料真正能支撑什么。

## 1. 来源与结构分工

读取 `sources/source-manifest.md` 的目的，是知道 Markdown 副本对应哪份原始材料、转换路线是什么、有没有可读边界。随后在结构抽取中为每个材料单元建立更细的位置参数：Markdown 起止行、章节标题、目录项、页码或等价定位。

来源记录负责“材料从哪里来、能不能读”；结构抽取负责“这一段材料在哪里、能支撑什么”。两者分开，规划师和创作者才能回到同一份材料。

## 2. 先创建结构抽取文件

确认材料可读后，先创建 `workspace/structure-extraction.md`，写入固定骨架和来源记录摘要。

“来源记录摘要”写入原始材料、Markdown 副本、转换路线和可读边界。文件先写入下面的固定骨架：

```markdown
# 结构抽取

## 来源记录摘要

- 原始材料：
- Markdown 副本：
- 转换路线：
- 可读边界：

## 材料单元列表

## 终极核心命题

## 论证地图

## KeyN / 关键命题

## 风险与边界

## 给规划师的提醒
```

结构抽取全程只写入 `workspace/structure-extraction.md`。

## 3. 划定材料单元

先保持 Markdown 来源完整，再划定后续工位能回到的材料单元。材料单元是理解颗粒，不是派工切块；它让判断能回到哪个章节、哪几行或哪一处等价位置。

划分时先确认“材料单元”是什么：它是一组相对完整的金字塔结构。每个单元应当有一个上层结论、中心判断或阶段性问题，下层有理由、解释、案例、数据、反例或推演来支撑它。后续规划需要拿着这个单元判断：这部分材料能支撑什么、不能支撑什么、可以放到哪一段认知路线里。

按下面顺序划分：

1. **书籍 / 电子书按章节划分**。书籍和电子书优先核对目录，目录项与正文章名、页码或标题对应时，直接按章节划分材料单元。
2. **再看材料内部的金字塔结构**。对论文、报告、文稿或标题较稳定的材料，读全文，用标题层级、摘要等辅助定位，但最终看这一组材料是否围绕同一个上层结论组织出足够支撑。只有当这组材料已经形成相对完整的上层结论和下层支撑关系时，才划为一个单元。
3. **最后处理无可靠格式的材料**。OCR 文本、格式异常材料或无可靠标题时，不相信误识别小标题、页眉页脚或样式碎片。直接全量读内容：以层级结构为入手点划分单元，当上层结论、中心问题、核心概念、论证对象、主要案例群或主要支撑组发生转换时，才是一个材料单元。

每个单元先只写清位置阐述：单元名称、位置参数和划分依据。位置参数可以是 Markdown 起止行、目录项、页码、标题层级或等价定位。划出单元后，在“材料单元列表”中，划定阶段先按这个格式写：

```markdown
### 单元 N：

- 位置参数：
- 划分依据：
```
注意事项：
1. 如果是规整的电子书材料，不优先读取全书，而是通过查找抽取整理出书籍目录结构。
2. 如果无固定的天然结构，先全量读取后根据材料划分单元。

## 4. 读取单元关键力量

逐个读取材料单元，每读完一个单元，就在 `workspace/structure-extraction.md` 的对应单元下补完核心命题、价值、颠覆、支撑、边界和后续用途。

每个单元至少回答：

- 核心命题：这部分真正回答什么问题。
- 价值：它提供哪一层新理解。
- 颠覆：它修正什么常见看法。
- 支撑：关键证据、故事、案例、研究、经验或论证。
- 边界：它没有证明什么，不能说满什么。
- 后续用途：更适合做开场、建架、推进、反例、行动，还是边界提醒。

注意事项：
1. 这里的语言要平衡洞察力与克制。材料先保持自己的立场，后面的规划再决定怎样面向顾客。
2. 有稳定结构的书籍，不要一次性全量读取全书，而是根据材料单元的位置参数读取后分析记录。
3. 如果无稳定结构，且在“划定材料单元”阶段已经做材料的全量读取，可以直接进行分析、记录。

## 5. 折出整体结构

单元读完后，再写整体判断：

- 终极核心命题：全材料反复折叠、不断回到的意义源头的那个命题中心。
- 论证地图：材料怎样围绕核心命题递进、转折、反驳、补充或收束。
- KeyN / 关键命题：真正反转俗见、能撑起后续规划和创作的关键判断。
- 风险与边界：材料不能外推、需要保守使用或需要补证的位置。

整体判断写入“终极核心命题”“论证地图”“KeyN / 关键命题”“风险与边界”。给后续工位的使用提示写入“给规划师的提醒”，说明哪些材料适合开场、建架、推进、反例、行动或边界提醒。

整体判断要从材料单元里长出来。它可以重新排列材料的意义关系，但不能把策展判断写成原文结论，也不能用流畅总论替代证据链。

## 6. 出站文件

出站时，`workspace/structure-extraction.md` 应包含：

```markdown
# 结构抽取

## 来源记录摘要

## 材料单元列表

每个单元写清：位置参数、划分依据、核心命题、价值、颠覆、支撑、边界、后续用途。

## 终极核心命题

## 论证地图

## KeyN / 关键命题

## 风险与边界

## 给规划师的提醒
```
